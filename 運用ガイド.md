---

## 1. これは何か

小学1・2年生の漢字の「同じ読み／同音異義語」を、**例文で意味を理解して正しく使い分ける**ことを学ぶ、単一HTMLのクイズアプリ。スマホ・タブレット・PCのブラウザで動作。

- 問題数: **約163問**（使い分け 33問 ＋ 同じ読み 130問）
- 漢字DB: **164字**（1年生80字＋2年生で使う字）

## 5. これまでの作業の流れ（セッション履歴）

1. **企画**: 小2漢字の同音異義語クイズのアイデア出し。配当漢字（1年80字／2年160字）を確認。
2. **試作**: 単一HTMLでクイズを作成。「使い分け」「同じ読み」の2モード、絵文字フィードバック・星評価。
3. **タイトル/表記調整**: タイトルを「同じ読みの かん字クイズ」に、サブを「― 文を読んで、正しいかん字をえらぼう ―」に。ふりがな（ruby）撤去。
4. **学年選択＋DB化**: 学年セレクター（1年/2年/1・2年）を追加。漢字情報を `KANJI` データベース（学年・画数・音訓）に集約し、出題を自動フィルタ。
5. **操作性**: 🏠 HOME ボタン、← 前へ ボタンを追加。回答状態を問題ごとに保持（戻っても二重採点しない）。出題順・選択肢をランダム化。
6. **問題拡充**: 同じ読みグループを多数追加し約163問に。使い分けも会う/合う・明ける/空ける・丸い/円いで33問へ（1・2年の自然な上限）。
7. **公開**: GitHub Pages で公開（手順は第6章）。
8. **整理**: 本体を `~/kanji-quiz/index.html` に一本化、`deploy.sh`・ドキュメント整備、Drive重複を削除。
9. **強化 第1弾**: 出題数選択（10/20/ぜんぶ）、まちがい復習（結果に誤答一覧＋「まちがえた問題だけ もう一回」）、解説に「完成した文」と「漢字メモ（画数・音訓）」を追加、空欄のaria・選択肢順の固定・キーボード操作（数字/Enter/←→）・フォーカス可視化。`validate.mjs`＋GitHub Actions で問題データを自動検証。（※音声読み上げは検討の結果不採用）
10. **強化 第2弾（ゲーム性・継続）**: localStorage によるセーブ層を追加。
    - **漢字ずかん**：正解した漢字がカード（画数・音訓付き）として図鑑に増える。1回正解で図鑑入り→累計3回で★マスター。図鑑対象は「出題の正解になる漢字」のみ（全112字）。
    - **連続正解コンボ**：れんぞく正解でコンボ表示＋紙ふぶき演出。
    - **メダル**：8種（はじめの一歩／全問正解／コンボ5・10／ずかん10・50字／マスター10字／ずかんコンプリート）。達成で自動解放。
    - ホームに「📖 かん字ずかん」「🏅 メダル」入口、結果画面に最高コンボ・新規メダル表示。
11. **強化 第3弾（メダル拡張・デイリー・復習）**:
    - **メダルを20種に拡張**（あそぶ回数／全問正解回数／累計正解数100・500／コンボ5・10・20／ずかん10・50・100・コンプ／マスター10・50・全／まちがいなおし／デイリー3日・7日連続）。
    - **デイリーチャレンジ**：日付をタネにした決定的シャッフルで「その日は全員・何回でも同じ10問」。クリアで**連続日数（ストリーク）**を加算（ホームに🔥表示）。
    - **まちがい復習（横断）**：まちがえた問題を localStorage に蓄積し、ホームの「🔁 まちがい ふくしゅう」からいつでも再挑戦。正解すると復習リストから自動で消える（克服＝comeback）。
    - 追加のセーブ項目：totalCorrect / perfectCount / comebacks / wrong{} / daily{streak,bestStreak,lastDay}。問題IDは BANK の index。
12. **強化 第4弾（3年生の解放・記録の保持）**:
    - **3年生を追加（解放制）**：ふだんは「3年🔒」。**1・2年の漢字を図鑑コンプリート（112字）で解放**。解放後は学年セレクターで「3年」が選べ、デイリーや「ぜんぶ」にも3年生が混ざる。メダル「🔓 3年生かいきん」も追加。
    - 3年生コンテンツ初期セット：使い分け **開ける/空ける/明ける・速い/早い・帰る/返る・登る/上る・追う/負う**、同じ読み **ちゅう（注/柱/中）**。DBに3年生漢字（開・速・返・登・追・負・注・柱）を追加。
    - 学年判定を一般化（`gradeSpec`：'1''2''3''all'＝解放状況で1・2 or 1〜3）。`validate.mjs` は学年1〜6を許可。
    - **記録の保持（仕組み化）**：localストレージに加え、ホーム下部に「💾 きろくを ほぞん／📂 よみこむ」を追加（JSONバックアップ＝端末移動・キャッシュ消去対策）。セーブに `version` を持たせ将来の移行に備える。
13. **3年生「同じ読み」拡充**: グループを追加（し=死/始/指/歯/使、とう=島/豆/湯/投、こう=港/向/幸、あん=安/暗、ひ=皮/悲、び=美/鼻、はな=鼻/花、よう=羊/葉/洋、やく=薬/役、どう=動、ちょう=調、きょく=曲、たい=待）。3年生の同じ読みは約27問に。問題数203・漢字DB203字。

## 6. ゼロから公開する手順（再現用メモ）

別マシンや別リポジトリで同じことをする場合の手順。

```bash
# 1. 公開用フォルダを用意（Google Drive内は避ける）
mkdir -p ~/kanji-quiz && cd ~/kanji-quiz
cp /path/to/quiz.html index.html        # URLを綺麗にするため index.html にする

# 2. Gitリポジトリ化して初回コミット
git init -b main
git config user.name  "あなたの名前"
git config user.email "you@example.com"
git add -A && git commit -m "初版"

# 3. GitHub公開リポジトリを作成してアップロード（gh CLI）
gh repo create kanji-quiz --public --source=. --remote=origin --push

# 4. GitHub Pages を有効化（mainのルートを公開）
gh api --method POST /repos/<ユーザー名>/kanji-quiz/pages \
  -f "source[branch]=main" -f "source[path]=/"

# 5. ビルド状況の確認 → URL
gh api /repos/<ユーザー名>/kanji-quiz/pages/builds/latest --jq .status   # built になればOK
gh api /repos/<ユーザー名>/kanji-quiz/pages --jq .html_url
```

> GitHub Pages の無料公開はリポジトリを **public** にする必要がある（本アプリに秘密情報はないので問題なし）。
> ※ ウェブ画面で行う場合: リポジトリ → Settings → Pages → Source を「Deploy from a branch」、Branch を `main` / `/ (root)` に設定。

## 7. メモ・今後の選択肢

- **動作確認の自動チェック**: 「答えが選択肢に含まれる／選択肢の漢字がDBにある／全選択肢に意味がある」を Node で検証可能（過去に全問パス）。
- **問題をさらに増やすには**: いまの同じ読みグループに別例文を足すのが手堅い。「使い分け」を大幅に増やすには3年生以上の漢字（暑い/熱い/厚い、開ける、速い/早い 等）の解禁が必要。
- **拡張アイデア**: 正解時に画数・音訓を表示／2年生160字を全部DB投入／1プレイの問題数を選択／効果音・イラスト・PWA化（オフライン対応）。
