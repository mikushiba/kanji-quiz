## 1. これは何か

小学1・2年生の漢字の「同じ読み／同音異義語」を、**例文で意味を理解して正しく使い分ける**ことを学ぶ、単一HTMLのクイズアプリ。スマホ・タブレット・PCのブラウザで動作。

- 問題数: **約163問**（使い分け 33問 ＋ 同じ読み 130問）
- 漢字DB: **164字**（1年生80字＋2年生で使う字）

25. **効果音＋3年生 解放ヒント（家族フィードバック対応）**:
    - **効果音を追加（Web Audio・音声ファイル不要／オフラインでも鳴る）**：`playSound()` を `choose()`（正解/不正解）と `confettiBurst()`（クラッカー）に組み込み。音はすべてコードで合成（オシレーター＋ノイズ）。
      - **正解＝ドアベル風「ピン♪ ポーン♪」**：E6(1318Hz)→C6(1047Hz) の2音に倍音を重ね、残響を長めにして鐘らしく。
      - **不正解＝番組風の濁ったブザー「ブッ ブー」**：半音差の2音（233+220Hz→220+207Hz）を sawtooth で重ね、わざと不協和に。2連で鳴らす。
      - **クラッカー＝「パン！」＋キラキラ**：破裂ノイズ（バッファ）＋上昇する3音。コンボ／全問正解／解放など紙ふぶきが出る場面で共通。
      - ※家族フィードバックで音色を2回微調整（正解の鐘・不正解のブザー）。`tone(freq,start,dur,type,vol)` ヘルパで各音を定義しているので、周波数・長さ・波形を変えれば調整可能。
    - **音声 ON/OFF 切替**：ホーム右上に「🔊 おと ON／🔇 おと OFF」トグル。設定は `store.soundOn` で localStorage に保存。OFF中は全効果音を抑止。
    - **3年生まで あと何字を表示**：ホームの学年セレクター下に「🔒 3年生まで あと N字（have/total）」を常時表示（解放後は「✅ 3年生も あそべるよ！」）。ロック中に「3年」を押したときの注意文にも残り字数を表示。

28. **4・5・6年生の枠を新設（DBは空のまま）**：
    - 学年ボタンを 1〜6年＋ぜんぶ に拡張し、`#seg` を**4列グリッド**で折り返し表示。
    - 解放ロジックを3年専用から**3〜6年に一般化**（`MAXGRADE=6`／`gradeUnlocked(g)`／`maxUnlockedGrade()`／`gradeHasContent(g)`／`collectiblesUpTo(g)`）。**前の学年までを図鑑コンプリートで順番に解放**。解放フラグは `store.unlocked3..6`。`checkUnlock()` は新規解放した学年番号を返す（結果バナーも「N年生が かいきん！」に対応）。
    - **問題がまだ無い学年（4〜6年）は「じゅんびちゅう」**として常にロック表示。タップすると「じゅんびちゅう」案内。`gradeSpec()`・`startDaily()`・`ぜんぶ` は解放済み学年のみ対象。
    - **後から `KANJI`/`BANK` に各学年を足すだけ**で、その学年は自動的に解放チェーン（前学年コンプリートで解放）に組み込まれる。`grade3Unlocked()` は `gradeUnlocked(3)` の別名として互換維持。
29. **3年生解放のデッドロック修正（家族フィードバック「あと20字から進捗が動かない／図鑑が埋まらない」）**：原因は、答えが1〜2年でも**選択肢に3年の字が混ざる問題**は「習っていない字は出さない」ルールで出題されず、その20字（音古楽力方玉昼多話前首書足雲頭風女人立六）が永久に集められず解放が詰んでいたこと。解放条件を「全COLLECTIBLE」ではなく**実際に集められる字だけ**で判定するヘルパー `collectableUpTo(g)`（選択肢が全てg学年以下の問題に出る正解漢字）を追加し、`gradeUnlocked()`・図鑑/ホームの「あと○字」表示・ロック時アラートをこれに差し替え。3年解放の必要数は143字に。解放後は「ぜんぶ」モードで残り20字も回収可能（解析で確認済み）。問題データは無改変・ロジックのみ。詳細な調査記録は `docs/2026-06-25-3年生解放デッドロック.md`。
30. **4年生の漢字DBを拡充（配当表どおり全202字）**：学年別漢字配当表（2017告示・2020全面実施）の4年配当202字をWikipedia等で網羅確認し、`KANJI` に `{g:4, s:画数, on:[音], kun:[訓]}` で全字追加（茨・媛・岡・潟・熊・鹿・栃・梨・崎・阪・阜など県名漢字を含む）。`tools/db-coverage.mjs` の `HAITOU` に4年リストを追記し、**4年: 202/202 完全**を確認。`validate.mjs` パス（KANJI 642字）。出題（BANK）は未追加で、4年生は引き続き「じゅんびちゅう」。
31. **4年生の「同じ読み」問題を一気に追加（183問）**：4年配当202字のうち、学年1〜4の字だけで「同じ読み」グループが組める **183字** を 答え に採用し、読み別グループ（あ〜ろ）で出題を作成（`BANK` 同じ読み 281→465）。すべて選択肢を **学年4以下** にそろえ（図鑑で集められる＝解放チェーンが詰まらない条件）、例文に答えの漢字を出さないルールを順守。`validate.mjs` 全パス（問題 515 / KANJI 642）。**残り19字（茨媛岡潟議熊芸材残順卒達栃梨末民浴陸類）は、学年1〜4の範囲に「同じ読み」の相手がいない**ため未出題（DBには収録済み）。今後 別形式（意味から選ぶ／5年生解禁後に相手字とペア）で回収予定。県名漢字は県名の読みで出題（鹿=鹿児島/佐=佐賀/埼=埼玉/崎=宮崎/滋=滋賀/縄=縄文/奈=奈良/阪=大阪/阜=岐阜/賀=年賀）。
32. **残り19字を新モード「いみ・かたちで えらぶ」(type:'imi') で回収＝4年生 202/202 完全**：同じ読みの相手が1〜4年にいない19字（茨媛岡潟議熊芸材残順卒達栃梨末民浴陸類）を、**選択肢をすべて学年4以下**にした `imi` 問題で出題（県名7字＋単独12字）。モードボタンを ①使い分け ②同じ読み **③いみ・かたち** ④ぜんぶ の4つに拡張（`startMode` は既存の `mode==='all'?true:q.type===mode` でそのまま動作）。`validate.mjs` を `imi` 許可に更新。**デッドロック検証スクリプトで、図鑑対象473字すべてが「ぜんぶ」モードで集められること・`collectableUpTo(4)` 全字が到達可能（5年解禁が詰まらない）ことを確認**。`imi` の選択肢は学年4以下なので、答えの19字も4年プレイ中に図鑑入りできる。問題 534（使い分け50/同じ読み465/いみ19）。

## 5.5 Lark Base 同期

`index.html` の `KANJI`（漢字DB）と `BANK`（問題）を Lark Base（多維表格）へ送る手段。

### 方法A：CSV を手動インポート（認証不要・すぐ）
```bash
cd ~/kanji-quiz && node -e '...'   # ※下記スクリプトで export/ に再生成
```
- 生成物：`~/kanji-quiz/export/kanji.csv`（270字）・`export/questions.csv`（261問）。UTF-8 BOM付き。
- Drive側 `共有ドライブ/family/kanji-quiz-export/` にもコピー済み。
- Lark Base →「インポート」→ CSVアップロードで取り込む。
- ※ `export/` は `.gitignore` 済み（リポジトリには入れない）。

### 方法B：API 自動同期（`lark-sync.mjs`）
`index.html` から読み取り、Lark Base のテーブルを「全削除→最新を一括追加」で置き換える（テーブル/列が無ければ自動作成）。

**準備（1回）**
1. Lark 開発者コンソールで**カスタムアプリ**を用意（既存アプリで可）。Base（多維表格）の**読み書き権限**を付け、公開/承認。
2. 同期先の **Base** を作り、そのアプリを**編集可コラボレーター**として追加。
3. `App ID` / `App Secret` / Base URL の `app_token`（`bascn...`）を控える。
4. `lark.config.sample.json` を `lark.config.json` にコピーして記入（**.gitignore 済み・ローカルのみ**）。
   - `domain`：Lark国際版 `https://open.larksuite.com` / Feishu(中国版) `https://open.feishu.cn`

**実行**
```bash
cd ~/kanji-quiz && node lark-sync.mjs
```
- 作成/更新テーブル：`漢字DB`（漢字・学年・画数・音読み・訓読み）/ `問題`（例文・ID・タイプ・分類・読み・正解・選択肢・選択肢の意味）。
- 何度でも実行可。問題を増やしたら再実行で最新化。`/schedule` で定期実行も可能。
- ⚠ 実機テストは認証情報待ちのため未実施（構文・データ抽出は確認済み）。初回に権限エラーが出たらスコープ名や `domain` を調整する。

### 同期データの定義（列）
- **漢字DB**: 漢字 / 学年(g) / 画数(s) / 音読み(on) / 訓読み(kun)
- **問題**: 例文(sentence, @@が空欄) / ID(BANK index) / タイプ(使い分け・同じ読み) / 分類(tag) / 読み(read) / 正解(answer) / 選択肢(choices) / 選択肢の意味(meaning)

## 6. ゼロから公開する手順（再現用メモ）

別マシンや別リポジトリで同じことをする場合の手順。

```bash
# 1. 公開用フォルダを用意（Google Drive内は避ける）
mkdir -p ~/kanji-quiz && cd ~/kanji-quiz
cp /path/to/quiz.html index.html        # URLを綺麗にするため index.html にする

# 2. Gitリポジトリ化して初回コミット
git init -b main
git config user.name  "あなたの名前"
git config user.email "you@example.com"
git add -A && git commit -m "初版"

# 3. GitHub公開リポジトリを作成してアップロード（gh CLI）
gh repo create kanji-quiz --public --source=. --remote=origin --push

# 4. GitHub Pages を有効化（mainのルートを公開）
gh api --method POST /repos/<ユーザー名>/kanji-quiz/pages \
  -f "source[branch]=main" -f "source[path]=/"

# 5. ビルド状況の確認 → URL
gh api /repos/<ユーザー名>/kanji-quiz/pages/builds/latest --jq .status   # built になればOK
gh api /repos/<ユーザー名>/kanji-quiz/pages --jq .html_url
```

> GitHub Pages の無料公開はリポジトリを **public** にする必要がある（本アプリに秘密情報はないので問題なし）。
> ※ ウェブ画面で行う場合: リポジトリ → Settings → Pages → Source を「Deploy from a branch」、Branch を `main` / `/ (root)` に設定。
