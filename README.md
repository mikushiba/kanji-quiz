# 同じ読みの かん字クイズ ― 仕様ドキュメント

小学1〜3年生の漢字の「同音異義語／同じ読みの漢字」を、**例文を通じて意味を理解し、正しく使い分ける**ことを目的とした学習クイズアプリ。

- **公開URL（トップ＝メニュー）**: https://mikushiba.github.io/kanji-quiz/ ／ 同じ読み= `…/onaji/` ／ 送りがな= `…/okurigana/`
- **リポジトリ**: https://github.com/mikushiba/kanji-quiz
- **構成（ポータル＋共有モジュール）**:
  - `index.html` … トップ（メニュー）
  - `onaji/index.html` … 同じ読み/使い分けクイズ本体（問題 `BANK` はここ）
  - `okurigana/index.html` … 送りがな練習（`KANJI` から自動生成）
  - `shared/kanji-db.js` … 漢字マスタ `KANJI`（**唯一の正データ・全アプリ共通**）＋`wordGrade`/`parseOkurigana`
  - `shared/kanji-ui.css` … 全アプリ共通のデザイン
- **ローカルで開く**: `open ~/kanji-quiz/index.html`（file:// でも共有DB/CSSを読める）

> このドキュメントの一部の説明は旧・単一HTML時代のもの。データ構造の `KANJI` は現在 `shared/kanji-db.js`、`BANK` は `onaji/index.html` にある。運用・公開・作業履歴は `運用ガイド.md` を参照。

## 1. 目的・コンセプト

- 同じ読みでも漢字によって意味が変わることを、**やさしい例文の中で**学ぶ。
- 「読み」だけでなく「意味」で漢字を選ぶ力を育てる。
- 低学年向けに、ひらがな多め・大きいボタン・すぐ出る解説・収集要素で構成。

---

## 2. 画面（全7種）

`showOnly(id)` で `home / quiz / result / dex / medals / help` を切り替え。

### ホーム（home）
- **❓ あそびかた**（子ども向けの使い方説明へ）
- **📅 きょうの チャレンジ**（デイリー。連続日数を表示）
- **学年セレクター**: 1年 / 2年 / 3年🔒 / ぜんぶ（3年は解放制）
- **もんだいの数**: 10 / 20 / ぜんぶ
- **モード①②③**（下記）
- **🔁 まちがい ふくしゅう**（のこり問題数を表示）
- **📖 かん字ずかん** / **🏅 メダル**
- **💾 きろくを ほぞん / 📂 よみこむ**（バックアップ）

### クイズ（quiz）
- 例文の□に入る漢字を選択肢から選ぶ。`← 前へ` / `🏠 HOME`、進捗バー、正解数、🔥コンボ表示。
- 回答すると即フィードバック（完成文＋全選択肢の意味＋漢字メモ）。

### 結果（result）
- 正解数・⭐評価・最高コンボ・新規メダル・**まちがえた問題の一覧**・「まちがえた問題だけ もう一回」。

### 図鑑（dex）/ メダル（medals）/ あそびかた（help）
- 後述。

### モード
1. **つかい分け（tsukai）** … 同じ音で意味が違う漢字を例文で見分ける（例: 会う/合う）
2. **同じ読み（onaji）** … 読みから正しい漢字を選ぶ（例: ほし＝星）
3. **ぜんぶ** … ①②をシャッフル

## 3. データ構造

### 3-1. `KANJI`（漢字マスタDB）
1文字ごとに学年・画数・音訓を持つ唯一の正データ。

```js
const KANJI = {
  '海':{ g:2, s:9, on:['カイ'], kun:['うみ'] },   // g:学年 s:画数 on:音[] kun:訓[]
};
```
- 1年80字＋2年・3年は出題で使う字を収録。学年判定・漢字メモ・図鑑はここから引く。

### 3-2. `BANK`（問題集）
```js
{
  type:'onaji',                    // 'tsukai' or 'onaji'
  tag:'同じ読み：かい',
  sentence:'夏は @@ で およぐ。',   // @@ が空らん
  read:'うみ',                     // @@ の読み
  answer:'海',                     // 正解（送りがな込み可）
  choices:['海','回','貝'],
  meaning:{ '海':'うみ','回':'まわる','貝':'かい（貝がら）' }
}
```
- 各問に `id`（BANKのindex）を起動時に付与（復習リスト用）。
- 学年は書かず、`KANJI[answer].g` から自動判定。

### 3-3. セーブ（localStorage / `kanjiQuiz.v1`）
```js
store = {
  version, kanji:{漢字:正解回数}, medals:{id:true}, wrong:{問題id:true},
  bestCombo, plays, perfect, perfectCount, totalCorrect, comebacks,
  daily:{streak,bestStreak,lastDay}, unlocked3
}
```

### 3-4. 学年フィルタ（`gradeSpec` / `gradeOK`）
- `currentGrade` は `'1' '2' '3' 'all'`。`'all'` は解放状況で `12`→`123` に展開。
- 選んだ学年の上限までの漢字しか選択肢に出さない＋正解の学年が対象か判定。

## 4. ゲーム性・継続（localStorageベース）

### 漢字ずかん（dex）
- 正解した漢字がカード（漢字・画数・音訓）として増える。
- **1回正解で図鑑入り → 累計3回正解で ★マスター**。
- 対象は「出題の正解になる漢字」。1・2年を全部集めると 3年生が解放。

### コンボ
- 連続正解で「🔥○れんぞく」表示、3・5・10…で紙ふぶき演出。最高コンボを記録。

### メダル（20種）
あそぶ回数（1/5/20）・全問正解・全問正解5回・累計正解100/500・コンボ5/10/20・まちがいなおし・ずかん10/50/100/コンプ・マスター10/50/全・デイリー3日/7日・**🔓3年生かいきん**。達成で自動解放、結果画面に新規表示。

### デイリーチャレンジ
- 日付をタネにした決定的シャッフルで「その日は同じ10問」。クリアで連続日数（ストリーク）加算。

### まちがい復習
- 間違えた問題を保存し、ホームからいつでも再挑戦。正解すると復習リストから消える（comeback）。

### 3年生の解放
- ふだんは「3年🔒」。**1・2年の図鑑コンプリート（118字）**で解放。解放後は「3年」選択や「ぜんぶ」「デイリー」に3年生が混ざる。

### 記録の保持（バックアップ）
- **💾 ほぞん**＝進捗をJSONでダウンロード、**📂 よみこむ**＝復元（確認ダイアログ付き）。
- セーブに `version` を持たせ将来の移行に備える。端末/ブラウザ間の移動・キャッシュ消去対策。

### あそびかた画面（子ども向け）
- やさしい日本語で、準備→モード→こたえ方→ボタン→収集→デイリー/復習→（保護者向け）バックアップを説明。

## 5. 収録コンテンツ（概要）

### 使い分け（同音異義・48問）
- 1・2年：あう（会う/合う）・あける（明ける/空ける）・まるい（丸い/円い）
- 3年：あける（開ける/空ける/明ける）・はやい（速い/早い）・かえる（帰る/返る）・のぼる（登る/上る）・おう（追う/負う）・きる（切る/着る）

### 同じ読み（213問）
読み（主に音読み）が同じ漢字グループから意味で選ぶ。1〜3年で多数のグループを収録（例：かい・こう・せい・とう・し・とう・しん・しゅう・しゅ・じ・けつ・らく・りょく・ほう・ひょう・がん・かん…）。

## 6. 編集・メンテナンス

### 問題を増やす
1. 新しい漢字を使うなら `KANJI` に追記（`{g,s,on,kun}`）。
2. `BANK` に問題を追加。`answer` は必ず `choices` に含める。
3. **既存の「同じ読み」グループは基本的に削除しない**（追加・拡充のみ）。
4. 検証：`node validate.mjs`（答えが選択肢にある／選択肢の漢字がDBにある／意味がある／学年1〜6 を自動チェック）。

### 公開へ反映（更新のしかた）
```bash
./deploy.sh "問題を追加"   # commit+push（GitHub Actionsで validate 自動実行）
```

### 道具（リポジトリ内 `tools/`）
- `tools/db-coverage.mjs` … 学年別配当表に対する網羅チェック（不足・学年ズレ）
- `tools/onaji-candidates.mjs` … これから作れる「同じ読み」グループ候補
- `tools/export-csv.mjs` … 漢字/問題のCSV書き出し（Lark/Sheets用）

### skill `kanji`（更新フロー）
- 編集→検証→公開→記録の標準フローを `/kanji` で実行。
- 置き場所は**利用フォルダ専用**（Googleドライブ `共有ドライブ/family/.claude/skills/kanji/`）。グローバルにもこのリポジトリにも置かない＝そのフォルダで起動したセッションだけで有効。
- 実体の手順は `cd ~/kanji-quiz` してから上記コマンドを実行する内容。

### 品質の自動化
- `validate.mjs`＋`.github/workflows/validate.yml`：push/PR時にデータ検証。実際に漢字のDB登録漏れ（古・区など）を公開前に検出した実績あり。
