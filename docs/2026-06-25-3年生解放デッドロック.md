# 3年生が解放されない不具合 — 調査・修正・検証の記録

- 日付: 2026-06-25
- 報告（家族フィードバック）: 「**3年生まであと20文字**の状態から進捗が一向に更新されない。カウントはちゃんと入っているのか? 図鑑がそもそも埋まらない。」
- 結論: **カウントは正しい。20字が仕様上どうやっても集められないデッドロックが原因。** ロジック修正で解消し公開済み。

---

## 1. 症状

- ホーム/図鑑の「3年生まで あと 20字」がずっと 20 から動かない。
- 図鑑（ずかん）が最後まで埋まらない。
- 結果として 3 年生がいつまでも解放されない。

## 2. 原因（デッドロック）

図鑑に集める対象＝**問題（BANK）の「正解」に出てくる漢字**（`COLLECTIBLE`）。1・2 年の対象は 163 字。
このうち **143 字は集められるが、残り 20 字は構造的に集められない**。

止まっていた 20 字:

> 音 古 楽 力 方 玉 昼 多 話 前 首 書 足 雲 頭 風 女 人 立 六

この 20 字は、それぞれ「正解になる問題が 1 問だけ」で、**その 1 問の選択肢に 3 年生の漢字が混ざっている**。

| 正解(1〜2年) | 問題の選択肢 | 邪魔している3年の字 |
|---|---|---|
| 人(1年) | 人 / **神** | 神 |
| 力(1年) | 力 / **緑** | 緑 |
| 女(1年) | 女 / **助** | 助 |
| 六(1年) | 六 / **緑** | 緑 |
| 楽(2年) | 楽 / **落** | 落 |
| 雲(2年) | 雲 / **運** | 運 |
| …（計20字、邪魔する字はすべて3年）|||

アプリには「**まだ習っていない学年の字は選択肢に出さない**」というルール（`gradeOK`）がある。
3 年がロック中だと選択肢の最大学年は 2 までに制限されるため、選択肢に 3 年の字を含むこれら 20 問は出題されない。

→ 20 字が永久に集まらない → 図鑑が埋まらない → 図鑑コンプリートが条件の **3 年解放が発火しない → 3 年の問題で集めることもできない**、という堂々巡り（デッドロック）。「あと 20 字」から 1 ミリも動かないのはこのため。

### 解放後は回収できることも確認

解析の結果、**3 年が一度解放されれば（「ぜんぶ」モードで）この 20 字はすべて回収可能**。
（選択肢の最大学年が 3 になり、20 問が出題対象に入るため。3 年図鑑コンプにも詰みなし。）
→ 噛み合っていないのは「解放の条件」だけ。

## 3. 修正（ロジックのみ・問題データは無改変）

`~/kanji-quiz/index.html`

解放条件を「全 `COLLECTIBLE`」ではなく「**いまのレベルで実際に集められる字だけ**」で判定するように変更。

追加ヘルパー:

```js
// g学年以下「だけ」で実際に集められる正解漢字
// （選択肢に上位学年を含む問題は出題されないので除外）
const collectableUpTo = (g)=> [...new Set(
  BANK.filter(q => q.choices.every(c => wordGrade(c) <= g))
      .flatMap(q => [...q.answer].filter(c => KANJI[c] && KANJI[c].g <= g))
)];
```

差し替えた箇所:
- `gradeUnlocked(g)` … 解放条件を `collectableUpTo(g-1)` で判定。
- 図鑑の「あと ○字（1・2年）で 3年生 かいきん！」表示（`renderDex`）。
- ホームの「○年生まで あと ○字（have/total）」表示（`refreshHome`）。
- 学年ボタンをロック中にタップしたときのアラート。

効果:
- **3 年解放に必要な数: 163 → 143 字**（集められない 20 字を条件から除外）。
- 143 字集めた時点で 3 年が解放され、その後ふつうに遊ぶうちに 20 字も埋まり、図鑑 100%（`dexAll`）も達成可能。

## 4. 検証

- `node validate.mjs` … 合格（問題数 331 / KANJI 440字）。
- CI（`validate.yml`, push: 3年生解放のデッドロック修正）… **completed / success**。
- 解析スクリプトで確認:
  - 「あと20字」の正体＝上記 20 字（すべて邪魔する distractor は 3 年）。
  - `collectableUpTo(2).length === 143`、143 字集めれば解放条件 true。
  - 3 年解放後（選択肢最大学年=3）に 20 字はすべて回収可能、3 年図鑑も詰みなし。

### 実セーブデータでの照合

家族から共有された記録 `かん字クイズ-きろく (1).json`（プレイ23回 / 累計666問正解）で照合:

| 項目 | 結果 |
|---|---|
| 修正後の解放に必要 | 143字 |
| 集めずみ | **143字（ちょうど全部）** |
| 残り | **0字 → 開き直せば即解放** |
| 旧基準の残り | 20字（＝集められない字そのもの） |

→ このお子さんは**集められる字を集めきっており、バグが無ければとっくに 3 年に到達していた**状態だった。
修正反映後にページを開き直せば、その時点で 3 年が自動解放される（これまでの記録は引き継がれる。セーブを読み込み直す必要はない）。

## 6. 教訓・今後の注意

- 「正解は配当内でも、選択肢（distractor）に上位学年の字を入れた問題」は、その学年がロック中は出題されない。
  → **その正解字が他に出題機会を持たない場合、図鑑・解放が詰む**。
- 4・5・6 年を `BANK` に足していくときも、同じ罠に注意。解放条件は `collectableUpTo()` ベースなので基本は自動で詰みを回避するが、
  「ある学年の字が、上位学年 distractor を含む問題にしか正解として現れない」状態は依然として図鑑コンプを妨げ得る。
  問題追加時は `node tools/db-coverage.mjs` 等で各学年の正解字が“素直に集められる問題”を1つ以上持つか確認するとよい。
