## 1. 4年生の漢字DB（202字）

- 出典: 学年別漢字配当表（2017告示・2020全面実施）。Wikipedia と漢字ドリルサイトの**2ソースを照合し完全一致**を確認（202字）。
- 県名漢字を含む現行版（茨・媛・岡・潟・熊・鹿・栃・梨・崎・阪・阜・賀・佐・埼・滋・縄・奈 など）。
- `KANJI` に `{g:4, s:画数, on:[音], kun:[訓]}` 形式で全字追加。`tools/db-coverage.mjs` の `HAITOU` に4年リストを追記 →**4年: 202/202 完全**。

## 2. 「同じ読み」問題（183字）

- 4年配当202字のうち、**学年1〜4の字だけで「同じ読み」グループが組める183字**を「答え」に採用。
- 例: かん(完・官・管・関・観) / き(希・季・旗・器・機・岐) / こう(功・好・候・康・香) / ふ(不・夫・付・府・阜・富) など。
- ルール順守:
  - 選択肢はすべて **学年4以下**（＝図鑑で集められる＝解放チェーンが詰まらない条件）。
  - **例文に答えの漢字を出さない**（`@@` のみ）。
  - 既存グループは削除せず追加のみ。
- 県名漢字は**県名の読み**で出題（鹿=鹿児島／佐=佐賀／埼=埼玉／崎=宮崎／滋=滋賀／縄=縄文／奈=奈良／阪=大阪／阜=岐阜／賀=年賀）。
- `BANK` 同じ読み 281 → **465**。

### 「同じ読み」で出せなかった19字
**茨 媛 岡 潟 議 熊 芸 材 残 順 卒 達 栃 梨 末 民 浴 陸 類**

理由: これらは **学年1〜4の範囲に「同じ読み」の相手がいない**（例: 陸=りく の相手が4年までにいない）。「同じ読み」は “読み→漢字” のクイズなので、相手がいないと作れない。

## 3. （撤回）`imi`モードの試作

- 一度、上記19字を新タイプ **`imi`（いみでえらぶ）** で出題（県名7字＋単独12字、選択肢は学年4以下）。
- ただし「モードが増えて分かりにくい」との家族判断で**撤回**。撤回しても19字はDBに残る（出題されないだけ＝デッドロックではない）。
- 撤回後、別案として **読みクイズ** を採用。

## 4. 「読みクイズ」モード（`type:'yomi'`）— 採用

### 4-1. なぜこれで解決するか
- 「同じ読み」は **読み→漢字**。対して読みクイズは **漢字→読み**（逆向き）。
- 漢字さえあれば出せるので、**読みの相手がいない19字も含め全字を出題・図鑑入り**できる。
- `KANJI` の `on`/`kun` が全字そろっているので、**手作りゼロ・全自動生成・全学年(1〜4年)対応**。低学年で空っぽにならない。

### 4-2. 仕様
- モードは ①使い分け ②同じ読み **③読みクイズ** ④ぜんぶ の4つ。`④ぜんぶ` は BANK＋読みクイズを混ぜる。
- 出題: 漢字を大きく表示「この かん字の 読みは？」→ 音/訓の読みを4択でえらぶ。
- 自動生成ヘルパー:
  - `kataToHira(s)` … 音読みカタカナ→ひらがな。
  - `yomiReadings(ch)` … その字の読み候補（音=ひら化／訓=送りがな込み `お(いる)→おいる`）。
  - `makeYomiQ(ch)` … 答え＝その字の読みの一つ、ダミー＝他字の読み（その字の読みと重複しないもの3つ）。`{type:'yomi', kanji, answer, choices, tag, id:'y'+ch}`。
  - `buildYomiPool()` … 選択中の学年の漢字ぶんを生成。
- 分岐実装: `startMode` / `render` / `choose` / `showFeedback` / 結果のまちがい一覧 で `type==='yomi'` を分岐。
  - 正解で **`q.kanji` を図鑑記録**（読みクイズは漢字を集める）。
  - 自動生成のため**復習リスト（`store.wrong`）には入れない**。

### 4-3. 図鑑（COLLECTIBLE）の拡張
- 旧: `COLLECTIBLE` = BANKの「正解」に出る漢字（473字）。
- 新: **`COLLECTIBLE` = 配当の全字（DB全642字）**。読みクイズで全字集められるので図鑑＝配当どおり。
- 未収集カードは「？ みつけよう」。集めると漢字＋画数・音訓が出る。

## 5. デッドロックにしない設計（重要）

- **解放ロジックは一切いじっていない**。学年解放の条件は従来どおり `collectableUpTo(g-1)`（＝**選択肢が全てg-1以下の問題に出る正解漢字**、BANK由来）。
- よって:
  - 既存の進行・ホーム/図鑑の「あと○字でかいきん」表示・既に解放済みのフラグ（`store.unlocked3..6`）に**影響なし**。再ロックされない。
  - 読みクイズで集めた字も `store.kanji` に入るので、**解放条件にもそのまま効く**（むしろ集めやすくなる）。
- 図鑑を全642字に広げても、解放は BANK由来の到達可能な字だけで判定するので**詰まない**。
- 検証（過去のシミュレーション）: 図鑑対象すべてが「ぜんぶ」モードで集められること、`collectableUpTo(4)` 全字が到達可能（5年解禁が詰まらない）ことを確認済み。

## 7. 触る場所（メンテ）

- 漢字を足す: `index.html` の `KANJI`（`{g,s,on,kun}`）。→ 読みクイズは自動で増える／図鑑にも自動で並ぶ。

## 8. 残課題・今後

## 9. 追記：全学年に「同じ読み／使い分け」を追加（+29問）

「1〜4年の字でもう追加できる問題はないか?」という問いに対し、`tools/onaji-candidates.mjs` 等で**まだ作れる未作成グループが残っている**ことを確認し、全学年にまんべんなく追加した。

### 9-1. 追加したグループ
- **同じ読み 8グループ**（別の音よみ＝「漢字に複数の読みがある」ことも学べる）
  - みょう（名・明・命）／ごう（合・強・業）／ふう（風・夫・富）／じつ（日・実）／け（家・化）／じき（食・直）／びょう（病・平）／ぎょう（行・業）
- **使い分け 6グループ**（同音で意味がちがう＝意味の使い分けを学ぶ）
  - あつい（暑い・熱い）／はかる（計る・量る・図る）／なおす（治す・直す）／かえる（変える・代える・帰る）／はやい（早い・速い）／つく（付く・着く）
- 答えの学年が **1〜4年に散る**よう選定。選択肢はすべて 学年4以下。
- 子ども向けに**不自然に難しい読みは除外**（悪=お／会=え／す=子数守主 など、共有読みが日常で出会いにくいもの）。

### 9-2. 「どのモード（学年）で出るか」の仕様（重要メモ）
問題が出題されるのは「**答えの漢字の学年**」のモード（`gradeOK`：選んだ学年＝答えの学年、かつ選択肢が全部その学年以下）。
- 例）熱・量・治・変・付・夫（答えが4年）→ **4年モード**に出る。
- 例）暑・計・早・速・代・帰（答えが1〜3年）→ その学年モードに出る。
- **「ぜんぶ」モードでは全部**出る。
- → 「4年生の問題を足す＝4年モードに出る」とは限らない。低学年の字が答えなら低学年が手厚くなる。

### 9-3. デッドロックは発生しない（理由と再検証）
- 解放条件は従来どおり `collectableUpTo(g)`（＝選択肢が全部g以下の問題に出る正解漢字＝**必ず集められる字だけ**）。**何問足してもこの不変条件は崩れない**。
- 追加分はすべて選択肢 学年4以下なので、なおさら安全。
- 再シミュレーション結果：`collectableUpTo(2)=144字`／`collectableUpTo(3)=277字`／`collectableUpTo(4)=461字`、**いずれも全字が「ぜんぶ」モードで集められる＝解放が詰まらない**ことを確認。
- 副作用メモ：選択肢が全部2年以下の新問題（じき：食・直）が増えたぶん、3年解放の必要数が 143→**144字**に +1（`食` が追加。よく出る字なので集めやすく、既に解放済みなら影響なし）。

### 9-4. 検証
- `node validate.mjs`: 全パス（**BANK 544：使い分け63／同じ読み481**、KANJI 642字）。
- デッドロック・シミュレーション: 上記9-3のとおり問題なし。
